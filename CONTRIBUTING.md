# Contributing to Epic Agent Fabric

Thank you for helping build the local-first, token-native agent OS.

## How to contribute

1. Fork the repo and create a feature branch
2. Follow the existing agent/skill patterns
3. Every change that lands on main must go through a Change Request (CR)
4. Update relevant skill docs and MEMORY.md when adding new capabilities
5. Test locally before opening a CR

## Code style

- Direct. Concrete. Cite file paths.
- One paragraph max on summaries.
- No emojis unless the user explicitly requests them.
- Match the user's tone.

## Security

- Never commit API keys or secrets
- All external access goes through the Executor Gateway
- New connectors must declare the narrowest possible scopes
- Memory edits require human review
